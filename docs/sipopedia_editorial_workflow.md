# Sipopedia Editorial Workflow (No Verbatim Policy)

## Non-Negotiable Rules
- Every published definition must be original Sipopedia editorial writing.
- Verbatim excerpts from books are not allowed.
- Each term must include:
  - `source_title`
  - `source_authors` (at least one)
  - `reference_links` (at least one)
  - `mla_citations` (at least one)
  - `purchase_links` (at least one)
- Keep `is_published = false` until editorial QA is complete.
- If using Amazon affiliate links, include this disclosure where purchase links appear:
  - `As an Amazon Associate I earn from qualifying purchases.`

## One-Book-At-A-Time Process
1. Pick one primary source book.
2. Build a candidate term list from that source.
3. Draft original definitions (no copy/paste prose from source).
4. Add practical application, examples, and related ideas.
5. Add MLA + reference links + purchase links.
6. Run editorial/legal QA.
7. Publish approved entries.

## QA Checklist Per Term
- No phrase-level copying from source prose.
- Definition is clear, accurate, and independent.
- Source metadata is complete.
- Citation format is consistent.
- Purchase links are valid and appropriate.
- Tone and structure match Sipopedia standards.
